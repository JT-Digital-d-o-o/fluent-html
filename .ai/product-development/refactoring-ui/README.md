# Refactoring UI

A systematic approach to interface design — start with function, define constrained systems, and let every design decision follow from a foundation of good structure. The goal: make interfaces feel "designed" without needing graphic design talent.

> **Precedence:** These are framework-agnostic design principles. Project-specific brand books override concrete choices (fonts, colors, spacing, shadows, components).

## Reading Order

1. [Design Process](design-process.md) — Start with features, systematize everything, work in cycles
2. [Visual Hierarchy](visual-hierarchy.md) — The most impactful tool: make important things look important
3. [Typography](typography.md) — Type scale, font selection, line height, alignment
4. [Color System](color-system.md) — HSL palette, shades, accessibility, warm/cool greys
5. [Spacing and Layout](spacing-and-layout.md) — Spacing scale, white space, layout principles
6. [Depth and Shadows](depth-and-shadows.md) — Light source, elevation system, flat depth techniques
7. [Images and Content](images-and-content.md) — Text over images, scaling, user-uploaded content
8. [Polish and Details](polish-and-details.md) — Finishing touches that separate good from great

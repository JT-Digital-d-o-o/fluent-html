# Polish and Finishing Details

The difference between "good" and "great" is in the finishing touches. You don't need graphic design talent — supercharge defaults, replace borders with subtler alternatives, and treat every component as an opportunity to go beyond the obvious.

---

## Supercharge the Defaults

Don't add new elements for flair — upgrade what's already there.

### DO: Replace generic elements with purposeful ones

| Default | Upgrade |
|---|---|
| Bullet points | Icons (checkmarks, arrows, or content-specific like padlock for security) |
| Quotation marks | Larger, colored quote marks as visual elements |
| Plain links | Custom underlines with color, thicker weight |
| Browser checkboxes/radios | Custom styled with brand colors |

Using brand colors for selected states instead of browser defaults takes a design from boring to polished.

---

## Add Color with Accent Borders

A colored rectangle requires no graphic design talent and goes a long way toward making something feel "designed".

### DO: Use accent borders in these positions

- Across the top of a card
- Highlighting active navigation items
- Along the side of an alert message
- As a short accent underneath a headline
- Across the top of the entire layout

---

## Decorate Backgrounds

When hierarchy, spacing, and typography are solid but the design still feels plain, add excitement to backgrounds.

### Options

- **Change background color**: emphasize panels or distinguish page sections
- **Subtle gradient**: use two hues no more than ~30° apart for an energetic look
- **Repeating pattern**: keep contrast low between pattern and background. Works along a single edge too
- **Individual shapes/illustrations**: geometric shapes, simplified graphics. Keep contrast low so nothing interferes with content

---

## Empty States Are a Priority

An empty state is a user's first interaction with a new feature. Don't waste it.

### DON'T: Show a blank page with chrome

No point presenting tabs, filters, or actions that don't do anything until the user creates content.

### DO: Design empty states deliberately

- Include an image or illustration to grab attention
- Emphasize the call-to-action to encourage the next step
- Hide supporting UI that serves no purpose yet
- Use the opportunity to be interesting and exciting

---

## Use Fewer Borders

Borders are the most common way to create separation, but too many make a design feel busy and cluttered.

### Alternative 1: Box shadow

Outlines an element like a border but more subtly. Works best when the element color differs from the background.

### Alternative 2: Different background colors

Adjacent elements with slightly different backgrounds create natural distinction. If you're already using different backgrounds plus a border, try removing the border.

### Alternative 3: Extra spacing

Simply increase the separation between groups. Creates distinction without introducing any new visual elements.

---

## Think Outside the Box

Don't let preconceptions limit how components look. Standard patterns are a starting point, not a constraint.

### Dropdowns

Break into sections, use multiple columns, add supporting text, include colorful icons — not just a white box with stacked links.

### Tables

Combine related columns when sorting isn't needed. Add images, introduce hierarchy within cells, use color to enrich data. A table doesn't have to be plain text in rigid columns.

### Radio buttons

For important UI choices, replace the standard circle-and-label with selectable cards — richer visual treatment that matches the importance of the decision.

### The principle

Constraints are powerful, but sometimes a bit of freedom is what takes an interface to the next level. Question every assumption about how a component "should" look.

# Lean Startup

Every product decision is an experiment. Validate the problem before building, build the smallest thing that tests your hypothesis, measure against pre-set targets, and pivot or persevere based on evidence — not intuition.

## Reading Order

1. [Lean Startup Manifesto](lean-startup-manifesto.md) — Build-Measure-Learn loop, value vs. waste, validated learning, experiment design
2. [MVP Types](mvp-types.md) — Smoke test, concierge, wizard of oz, single-feature: pick the fastest way to learn
3. [Pivot or Persevere](pivot-or-persevere.md) — Pivot taxonomy, decision triggers, the pivot meeting
4. [Small Batches](small-batches.md) — Optimize for learning speed, deployable slices, the envelope-stuffing principle

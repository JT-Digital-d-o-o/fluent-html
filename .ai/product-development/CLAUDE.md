# Product Development Guidelines

Start with [README.md](README.md) for the full table of contents and cross-cutting concept index.

## Precedence

Refactoring UI provides general design principles. For JT Digital projects, the [Brand Book](../brand-book/CLAUDE.md) overrides specific choices (fonts, colors, components, spacing, shadows).

## Decision Order

When making product decisions, consult guidelines in this order:

1. **lean-startup/** for whether to build it (validate the hypothesis)
2. **shape-up/** for how to scope and ship it (appetite, shaping, betting, building)
3. **refactoring-ui/** for how to design it (hierarchy, systems, polish)

## Lookup

- Use the cross-cutting index in README.md to find all files relevant to a specific concept (e.g., "appetite" spans 3 files)
- Each category has its own README.md with reading order and key concepts
- Each guideline file uses DO/DON'T pairs as atomic units — search for "DO:" or "DON'T:" to find specific rules
